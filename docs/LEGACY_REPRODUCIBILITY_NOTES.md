# Reproducibility Guide

## Environment

The reference run used Windows 11, Python 3.13.9, PyTorch 2.11.0 with CUDA 12.8, and an NVIDIA GeForce RTX 5080. Install the Python dependencies listed in `requirements-research.txt`. PyTorch installation may need the CUDA-specific wheel index appropriate for the target machine.

## Data Provenance

- Thebe Gas Field expert labels: Harvard Dataverse DOI `10.7910/DVN/YBYGBK`, CC BY 4.0.
- CRACKS v2: Zenodo DOI `10.5281/zenodo.13926822`, CC BY 4.0.
- F3: Netherlands Offshore F3 public OpendTect survey; local sparse FaultA interpretation is used only under the documented stick split.
- Synthetic volumes: generated by the scripts and configuration in this workspace.

Do not redistribute third-party data without retaining its original license and citation. The public repository should provide download and preparation commands rather than silently repackaging source archives.

## Key Commands

Prepare Thebe data:

```powershell
python -m fault_experiments.prepare_thebe_dataset --splits train,val,test
```

Rebuild frozen Thebe statistics and figures from completed predictions:

```powershell
python -m fault_experiments.summarize_thebe_final_test
python -m fault_experiments.plot_thebe_final_statistics
python -m fault_experiments.plot_cross_dataset_evidence
```

Prepare CRACKS and reproduce the preregistered audit. The evaluator reads audit masks only and does not load `sealed_reserve`:

```powershell
python -m fault_experiments.prepare_cracks_external
python -m fault_experiments.evaluate_cracks_audit
python -m fault_experiments.summarize_cracks_audit
python -m fault_experiments.visualize_cracks_audit
```

Rebuild the artifact hash and environment manifest:

```powershell
python -m fault_experiments.export_reproducibility_manifest
```

## Fixed Evaluation Policy

- U-Net threshold: 0.50.
- Hybrid DSA threshold: 0.15.
- Matched SwinUNETR F3-chain threshold: 0.40.
- Thresholds were selected only on Thebe `val1-val2`.
- Thebe `test2-test7` is never used for threshold or checkpoint selection.
- CRACKS audit has no dataset-specific calibration.
- CRACKS reserve was opened once after `FINAL_PROTOCOL_LOCK.json` fixed checkpoints, thresholds, and metrics. Results are in `runs/cracks_reserve_final`; rerunning the reserve evaluator is intentionally refused when that output exists.

## Known Environment Limitation

`torch.compile` with the Inductor backend was unavailable on the reference Windows environment because no working Triton installation was present. Eager and CUDA Graph timings are reported separately; CUDA Graph did not materially improve Hybrid DSA latency.
